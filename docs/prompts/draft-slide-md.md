# LLM向け原稿作成プロンプト集

MD Slide Studio のスライドMDをLLM（Claude / ChatGPT / Gemini 等）に書かせるための
プロンプト例集。アプリ内の「🤖 AIプロンプト」ボタンからも基本形をコピーできます。

## 1. 基本形（新規作成）

アプリ内の🤖ボタンと同一のプロンプト（`src/ai/draftAssistPrompt.ts`）。
テーマを差し替えて使う。

## 2. 既存メモ・議事録からの変換

```
以下のメモを、スライドMD仕様（基本形プロンプトの仕様セクションを添付）に準拠した
6〜8枚のスライドMDに構造化してください。

変換ルール:
- 箇条書き → points、表 → table、数値データ → chart、手順 → summary
- 1スライドの分量は「情報グループ優先」で決める（見出し単位で自然に分割）
- 数値をグラフ化する場合、メモに単位・出典が揃っているものだけを chart にする
  （揃わない数値は表かテキストのまま残す）
- 最終スライドは sources（出典が無ければメモの出所を書く）

# メモ
（ここに貼り付け）
```

## 3. スタイル・レイアウトの調整指示

生成済みMDに対する修正依頼の例:

```
このスライドMDの3枚目を2カラムにしてください。
ディレクティブを <!-- slide: points, layout: two-col --> に変更するだけで、
本文は変えないでください。
```

利用可能な layout: `two-col`（points/summary）/ `title-xl`（title）/ `compact`（points/table/summary）

パレット変更は frontmatter の `palette:` を
ocean / forest / sunset / plum / graphite のいずれかに変える。

## 4. レビュー依頼

```
以下のスライドMDをレビューしてください。観点:
1. 制約違反（枚数3〜12 / 1スライド1グラフ / 最終スライドがsources / 生HTML禁止）
2. グラフの数値に出典が無い・推定値が混ざっていないか
3. 1スライドの情報量が過多でないか（表10行超・リスト20項目超は fit 付与を提案）
修正版のMDを出力してください。

（ここにMDを貼り付け）
```
